| Term | Meaning |
|---|---|
| Boot memory | fixed context loaded at session start |
| Disclosure | the future situation where a memory should appear |
| Domain | top-level memory namespace such as `core`, `preferences`, or `project` |
| Glossary term | explicit keyword or phrase that helps recall |
| Memory path | stable URI for a memory |
| Recall | process that finds relevant memories for the current task |
| Dream | maintenance workflow that reviews and improves memory quality |
| Priority | importance level used to rank and protect memories |
