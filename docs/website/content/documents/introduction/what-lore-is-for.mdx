Lore is useful for durable context:

| You want to keep | Example |
|---|---|
| User preferences | preferred name, timezone, working style, language choice |
| Project facts | repository paths, deployment URLs, architecture decisions |
| Operating rules | how to run tests, how to release, what must be checked first |
| Investigation results | root causes, decisions, follow-up constraints |
| Product knowledge | feature behavior, known tradeoffs, glossary terms |

A good memory answers a future question before the agent asks it again.
