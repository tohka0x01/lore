Dream changes should be judged by practical value:

| Check | Good sign |
|---|---|
| Scope | one memory change addresses one clear issue |
| Trigger | disclosure says when the memory should appear |
| Terms | glossary contains real user/project language |
| Safety | core boot memories stay stable |
| Traceability | diary explains the reason for the change |

If a Dream change makes recall worse, use the review and rollback flow from the Dream page.
