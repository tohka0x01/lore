选择与你日常工具匹配的运行时集成。

## 运行时选择

| 运行时 | 什么时候使用 | 集成行为 |
| --- | --- | --- |
| Claude Code | 你用 Claude Code 做编码会话 | MCP 工具、`SessionStart` 启动注入、每次 prompt 召回、`CLAUDE.md` guidance 导入 |
| Codex | 你用 Codex CLI/Desktop | MCP 工具、Lore skill、插件注册、hooks feature 和兼容的用户级 hooks |
| Pi | 你用 Pi agent | 扩展 hooks 注入启动 guidance 和每次 prompt 召回 |
| OpenClaw | 你用 OpenClaw | 插件工具、启动健康检查、启动 guidance 和召回注入 |
| Hermes | 你用 Hermes Agent | Python skill/provider，包含 boot、recall 和 memory tools |
| OpenCode *（预发布 `v1.3.15-pre.4`）* | 你使用 OpenCode 1.18.3 或更新版本 | 精确命名的原生 `lore_*` 工具、通过 `experimental.chat.system.transform` 注入 Boot、通过 `chat.message` 执行 prompt Recall |

只安装你使用的集成：

```bash
npx @loremem/cli install --channels codex,claudecode
```

只安装 OpenCode 预发布版：

```bash
npx @loremem/cli install --lang zh --pre --channels opencode
```

它把 `lore-opencode.zip` 安装为 `~/.config/opencode/plugins/lore-memory.js`，并与其他客户端共用 `~/.lore/config.json`。原生工具列表从 `lore_guidance` 到 `lore_move_node`。System 注入 fail open；仅实验性 system hook 不可用时，prompt Recall 仍会继续。OpenCode 标准安装不配置 MCP；`/api/mcp` 仅作为手动兜底。

每个 channel 都应该使用同一个服务器 URL 和 token。基于 MCP 的客户端端点包含 `client_type` 参数；原生 OpenCode REST 请求以 `opencode` 标识自己，不会添加 MCP 配置。
